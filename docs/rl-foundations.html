Reinforcement Learning Foundations

Lectures: L12–L13 • Prince: Chapter 19

Overview

RL is a sequential decision-making framework. An agent interacts with an environment to maximize cumulative reward.

At each time step $t$:

  1. Agent observes state $s_t$
  2. Agent takes action $a_t$
  3. Environment returns reward $r_{t+1}$ and next state $s_{t+1}$

Markov Decision Processes

An MDP is defined by the tuple $(S, A, \Pr(s_{t+1}, r_{t+1} \mid s_t, a_t), \gamma)$:

ComponentMeaning
$S$State space
$A$Action space
$\Pr(s', r \mid s, a)$Transition dynamics (may be unknown)
$\gamma \in (0, 1]$Discount factor (larger = more far-sighted)
Markov Property: The future depends only on the current state, not on the full history: $\Pr(s_{t+1} \mid s_t, a_t) = \Pr(s_{t+1} \mid s_1, a_1, \ldots, s_t, a_t)$.

Building Blocks: MP → MRP → MDP

  • Markov Process (MP): states + transitions only (no actions, no rewards)
  • Markov Reward Process (MRP): add reward function $r[s]$
  • MDP: add actions — agent can influence transitions

Return and Discount

The return from time $t$ is the discounted sum of future rewards:

$$G_t = \sum_{k=0}^{\infty} \gamma^k \, r_{t+k+1}$$

$\gamma < 1$ ensures the sum converges and makes the agent prefer sooner rewards.

Policy

A policy $\pi$ dictates the agent's behavior:

  • Deterministic: $a = \pi(s)$
  • Stochastic: $\pi[a \mid s] = \Pr(a \mid s)$

Goal: find the optimal policy $\pi^*$ that maximizes expected return.

Value Functions

State-Value Function

$$v[s \mid \pi] = \mathbb{E}_\pi[G_t \mid s_t = s]$$

Expected return when starting in state $s$ and following policy $\pi$.

Action-Value Function

$$q[s, a \mid \pi] = \mathbb{E}_\pi[G_t \mid s_t = s, a_t = a]$$

Expected return when starting in $s$, taking action $a$, then following $\pi$.

Relationships Between $v$ and $q$

$$v[s] = \sum_a \pi[a \mid s] \cdot q[s, a] \quad \text{(v from q: average over policy)}$$
$$q[s, a] = r[s, a] + \gamma \sum_{s'} \Pr(s' \mid s, a) \, v[s'] \quad \text{(q from v: one-step lookahead)}$$

Bellman Equations

Self-consistency constraints that any value function must satisfy:

Bellman Equation for $q$

$$q[s_t, a_t] = r[s_t, a_t] + \gamma \, \mathbb{E}_{s_{t+1}} \left[ \mathbb{E}_{a_{t+1} \sim \pi} \left[ q[s_{t+1}, a_{t+1}] \right] \right]$$

In words: the value of taking action $a$ in state $s$ equals (1) the immediate reward, plus (2) the discounted expected value of the next state-action pair.

Bellman Optimality Equation

For the optimal policy, replace "expected action under $\pi$" with "best action":

$$q^*[s, a] = r[s, a] + \gamma \sum_{s'} \Pr(s' \mid s, a) \max_{a'} q^*[s', a']$$
Key Insight

The Bellman equations create a "ripple effect": changing $v[s]$ affects values of predecessor states. Many RL algorithms are methods to solve or approximate these equations.

Exploration vs Exploitation

  • Exploitation: choose actions that currently look best
  • Exploration: try actions that might reveal higher long-term return
  • You cannot learn $q(s,a)$ for actions you never take!

Exploration Strategies

StrategyDescription
$\epsilon$-greedyWith probability $\epsilon$ take random action, else greedy
Softmax/BoltzmannSample proportional to $\exp(q/\tau)$
Optimistic initializationStart $q$ high to encourage trying all actions
