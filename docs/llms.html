LLMs & Decoding Strategies

Lecture: L11 • Prince: Chapter 12 (partial)

Decoder-Only Transformers

The most common LLM architecture (GPT, LLaMA, Mixtral). Each token attends only to itself and previous tokens via causal masking. The final token's contextualized embedding is mapped to a probability distribution over the vocabulary.

From Vectors to Words

The transformer outputs a vector for the final position. An MLP (or single linear layer) + softmax converts this to a probability distribution over the vocabulary:

$$P(w_i) = \text{softmax}(\mathbf{W}_\text{head} \cdot \mathbf{h}_\text{final} + \mathbf{b})_i$$

Decoding Strategies

Greedy Search (Argmax)

Select the most probable token at each step. Simple and fast, but produces repetitive, predictable text.

Beam Search

Maintain $B$ candidate sequences (beams) at each step. Expand each by all vocabulary tokens, keep top-$B$ by cumulative probability. Produces more coherent output than greedy.

Temperature Sampling

Sharpen or flatten the distribution before sampling:

$$P(w_i) = \frac{e^{z_i / T}}{\sum_j e^{z_j / T}}$$
  • $T = 1$: standard sampling
  • $T < 1$: sharper distribution ⇒ more deterministic, coherent
  • $T > 1$: flatter distribution ⇒ more random, creative
  • $T \to 0$: converges to greedy (argmax)

Top-k Sampling

Restrict sampling to only the top $k$ most probable tokens. Reduces nonsensical outputs while maintaining diversity.

Top-p (Nucleus) Sampling

Choose the smallest set of tokens whose cumulative probability exceeds threshold $p$. More adaptive than top-k: narrows or widens dynamically based on the model's confidence.

Advanced Decoding

Speculative Decoding

  1. A smaller, faster model generates candidate tokens speculatively
  2. The large model verifies these tokens in parallel
  3. Accept correct tokens; regenerate from the first mistake

Significant speedup without compromising output quality.

Structured Decoding

Constrain the LLM to produce specific output formats (JSON, multiple choice, regex patterns) by filtering the vocabulary at each step. Libraries like outlines implement this.

Retrieval-Augmented Generation (RAG)

Combines retrieval with generation to address hallucinations and factual inaccuracy:

  1. Convert the query into an embedding
  2. Retrieve relevant documents via nearest-neighbor search in embedding space
  3. Prepend retrieved documents as context for the LLM

Retrieval is based on semantic similarity (embedding dot products), not just keyword matching.

Multimodal LLMs

Process multiple modalities (text, images, audio) within a single model by converting everything to tokens.

Vision Transformers (ViT)

Treat image patches as tokens. Flatten, embed, add positional encodings, process with transformer blocks.

CLIP (Contrastive Language-Image Pretraining)

Trained on image-text pairs using contrastive learning:

$$\mathcal{L} = -\frac{1}{2N}\sum_{i=1}^{N}\left[\log\frac{\exp(\mathbf{v}_i^\top \mathbf{u}_i)}{\sum_j \exp(\mathbf{v}_i^\top \mathbf{u}_j)} + \log\frac{\exp(\mathbf{v}_i^\top \mathbf{u}_i)}{\sum_j \exp(\mathbf{v}_j^\top \mathbf{u}_i)}\right]$$

where $\mathbf{v}_i$ = image embedding, $\mathbf{u}_i$ = text embedding. Matches images with their correct text descriptions in a shared space. Enables zero-shot classification: compare image embedding against text embeddings of candidate labels.
