Transformers

Lectures: L08–L10 • Prince: Chapter 12

Design Motivation

Originally designed for NLP. Three requirements:

  1. Compute efficiency: process hundreds/thousands of interconnected words encoded as high-dimensional vectors
  2. Length generalization: handle variable-length input sequences
  3. Attention: attend to specific parts of earlier text for context (possibly very far away)

Tokenization

Convert text to a sequence of integer indices, each mapped to a learned embedding vector.

Tokenization Levels

LevelProsCons
Word-levelSimple, intuitiveLarge vocabulary, OOV problems
Character-levelHandles any textVery long sequences, slow
Subword (BPE)Balance of granularity and efficiencyMore complex

Byte Pair Encoding (BPE)

  1. Start with character-level tokens
  2. Count pair frequencies across the corpus
  3. Merge the most frequent pair into a new token
  4. Repeat until desired vocabulary size

Example: "low lower lowest" → merge (l,o)→LO, (LO,w)→LOW → vocabulary = {LOW, , e, r, s, t}

Embeddings

Each token index is looked up in a learned embedding matrix to produce a dense vector. These embeddings capture semantic relationships in a continuous space.

Self-Attention

The core mechanism of transformers. For a sequence of vectors $\{\mathbf{x}_1, \ldots, \mathbf{x}_n\}$:

Queries, Keys, and Values

Each input vector $\mathbf{x}_i$ is projected into three different representations:

$$\mathbf{q}_i = \mathbf{W}_Q \mathbf{x}_i, \quad \mathbf{k}_i = \mathbf{W}_K \mathbf{x}_i, \quad \mathbf{v}_i = \mathbf{W}_V \mathbf{x}_i$$

Attention Weights

Compute how much each position $j$ should contribute to position $i$:

$$a_{ij} = \text{softmax}_j\!\left(\frac{\mathbf{q}_i \cdot \mathbf{k}_j}{\sqrt{d_k}}\right)$$

Divide by $\sqrt{d_k}$ to keep variance unit-scale (prevents softmax saturation).

Output

$$\text{output}_i = \sum_{j=1}^{n} a_{ij} \, \mathbf{v}_j$$
Intuition: Each position $i$ "queries" all positions $j$. The dot product $\mathbf{q}_i \cdot \mathbf{k}_j$ measures compatibility. The output is a weighted average of values, where the weights come from the query-key compatibility.

Matrix Form

$$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\!\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right)\mathbf{V}$$

Multi-Head Attention

Run $H$ independent attention heads in parallel, each with its own $\mathbf{W}_Q^h, \mathbf{W}_K^h, \mathbf{W}_V^h$, then concatenate and project:

$$\text{MultiHead}(\mathbf{X}) = \text{Concat}(\text{head}_1, \ldots, \text{head}_H) \cdot \mathbf{W}_O$$

Each head attends to different features/relationships (analogous to multiple CNN channels).

Positional Encodings

Self-attention has no built-in notion of position. Solution: add or concatenate position information to the embeddings.

Sinusoidal Positional Encodings

Fixed encodings using sine and cosine functions at different frequencies. The periodic nature captures relative positions, acting as a "ruler" for attention.

Learned Positional Embeddings

A separate embedding table mapping position indices to vectors. More flexible but limited to a maximum sequence length.

The Transformer Block

Each block applies (with residual connections and layer normalization):

x → LayerNorm → Multi-Head Attention → (+x) → LayerNorm → MLP (Feed-Forward) → (+x)

The MLP (feed-forward network) is applied independently to each position. It typically expands dimensions (e.g., 4×) and then contracts back.

ArchitectureTypeMaskingUse Case

Encoder-Only (BERT)

  • Bidirectional: each token can attend to all others
  • Pre-trained with masked language modeling (predict randomly masked tokens)
  • Fine-tuned on downstream tasks

Decoder-Only (GPT)

  • Autoregressive: each token can only attend to itself and previous tokens
  • Causal masking: set future attention weights to $-\infty$ before softmax
  • The final token's embedding is used for next-token prediction

Cross-Attention

Used in encoder-decoder models. The decoder's queries attend to the encoder's keys and values, allowing the decoder to condition on the full input sequence.

Vision Transformers (ViT)

  • Split images into fixed-size patches (e.g., 16×16)
  • Flatten each patch and embed it as a token
  • Add positional embeddings
  • Process through standard transformer blocks
  • Self-attention captures long-range dependencies across the entire image
