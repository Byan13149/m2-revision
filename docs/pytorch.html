PyTorch Core

Lecture: L06 (Notebook)

1. Tensors

PyTorch tensors are NumPy-like arrays with GPU support and automatic differentiation.

import torch
x = torch.tensor([5., 3.])           # Create a tensor
x = torch.randn(10)                   # Random normal
x = torch.zeros(10)                   # Zeros
x_np = x.numpy()                      # To NumPy
x_t = torch.tensor(np.array([1,2,3])) # From NumPy

2. Devices: CPU vs GPU

device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
x = x.to(device)  # Move tensor to GPU

3. Autograd: Automatic Differentiation

PyTorch performs tape-based reverse-mode autodiff. Set requires_grad=True to track gradients.

x = torch.tensor([np.pi/4], requires_grad=True)
y = torch.sum(torch.cos(x) ** 2)
grads = torch.autograd.grad(y, x)[0]  # Exact gradient

Each operation creates a grad_fn pointer that forms the computation graph for backpropagation.

4. Optimization Loop

w = torch.tensor([0.0], requires_grad=True)
opt = torch.optim.SGD([w], lr=0.1)

for step in range(20):
    opt.zero_grad()        # Clear old gradients
    loss = (w - 3.0).pow(2).sum()
    loss.backward()        # Compute gradients
    opt.step()             # Update parameters

5. nn.Module: Building Models

Subclass nn.Module for automatic gradient tracking and parameter management.

class DeepNN(nn.Module):
    def __init__(self, hidden=32):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(1, hidden),
            nn.ReLU(),
            nn.Linear(hidden, hidden),
            nn.ReLU(),
            nn.Linear(hidden, 1),
        )

    def forward(self, x):
        return self.net(x)
Important

All child modules must be initialized in __init__ (not in forward) so that PyTorch can discover and track their parameters.

6. Full Training Loop

net = DeepNN(hidden=32).to(device)
opt = torch.optim.Adam(net.parameters(), lr=1e-2)
loss_fn = nn.MSELoss()

for step in range(500):
    opt.zero_grad()
    x = torch.randn(256, 1, device=device)
    y = torch.sin(x)
    pred = net(x)
    loss = loss_fn(pred, y)
    loss.backward()
    opt.step()

8. MLP vs CNN Comparison

On MNIST-1D (a small 1D dataset designed to make architecture differences obvious):

ModelParametersTest Accuracy (20 epochs)
Simple MLP (1 hidden layer, 64 units)3,274~51%
Simple CNN (2 conv + 1 FC)1,738~67%

The CNN achieves much better accuracy with fewer parameters, demonstrating the power of inductive bias for structured data.

Key Patterns to Know

  • model.train() / model.eval() — switches dropout and batch norm behavior
  • with torch.no_grad(): — disables gradient tracking for inference
  • loss.backward() — computes gradients
  • optimizer.zero_grad() — clears accumulated gradients before each step
  • optimizer.step() — applies the gradient update
  • .unsqueeze(1) — add channel dimension for CNN input
