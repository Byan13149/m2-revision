Convolutional Neural Networks

Lectures: L04–L05 • Prince: Chapters 10–11

Motivation: Problems with Fully-Connected Networks for Images

  1. Size: A 224×224 RGB image = 150,528 dimensions. One FC hidden layer ⇒ ~22 billion weights
  2. Spatial structure: Nearby pixels are statistically related; FC networks ignore this
  3. Translation: The same object at different locations should be recognized identically

Invariance and Equivariance

Invariance: $f[t[\mathbf{x}]] = f[\mathbf{x}]$ — output unchanged by transformation. Example: image classification should be invariant to translation.

Equivariance: $f[t[\mathbf{x}]] = t[f[\mathbf{x}]]$ — output transforms the same way as input. Example: segmentation maps should shift when the image shifts.

Convolution is equivariant to translation by construction.

1D Convolution

Output is a weighted sum of neighbors using a kernel (filter):

$$y_i = \sum_{k=0}^{K-1} w_k \cdot x_{i+k} + b$$

where $K$ is the kernel size.

Key Parameters

ParameterEffect
Kernel size $K$Number of input positions combined per output. More parameters but larger receptive field.
Stride $s$Shift by $s$ positions per output. Stride > 1 decreases output size (downsampling).
Dilation $d$Intersperse kernel values with zeros. Larger receptive field without more parameters.
PaddingZero padding: pad input with zeros to maintain size. Valid: no padding, output is smaller.
Convention Warning

Always check convention! Sometimes dilation=1 means standard behavior (no dilation), sometimes dilation=0 does.

Convolution as Special Case of FC

A convolutional layer is a fully-connected layer with shared weights and sparse connections. The weight matrix has a banded (Toeplitz) structure with repeated entries.

Channels (Feature Maps)

A single convolution loses information. Solution: apply multiple convolutions and stack the results as channels.

Parameter Count

$$\text{Parameters} = C_\text{out} \times (C_\text{in} \times K + 1)$$

For 2D convolution with kernel $K \times K$:

$$\text{Parameters} = C_\text{out} \times (C_\text{in} \times K^2 + 1)$$

Receptive Fields

The receptive field of a unit is the region of the input that influences its value. With deeper layers, the receptive field grows, allowing the network to capture larger-scale patterns while keeping kernels small.

2D Convolution

Same principles as 1D, but the kernel is $K \times K$. Weighted sum over a 2D region:

$$y_{i,j} = \sum_{m=0}^{K-1}\sum_{n=0}^{K-1} w_{m,n} \cdot x_{i+m, j+n} + b$$

Downsampling and Upsampling

Downsampling Methods

  • Strided convolution: stride > 1 (e.g., stride 2 halves each spatial dimension)
  • Max pooling: take maximum over a window. Provides partial translation invariance.
  • Mean pooling: take average over a window

Upsampling Methods

  • Nearest-neighbor: duplicate pixels
  • Bilinear interpolation: weighted average of neighbors
  • Max-unpooling: place value at remembered max position
  • Transposed convolution: learnable upsampling (inserts zeros, then convolves)

1×1 Convolution

Mixes channels without spatial interaction. Equivalent to applying a fully-connected layer independently at each spatial position. Can change the number of channels.

Key Architectures

AlexNet (2012)

  • 60 million parameters (most in FC layers)
  • Data augmentation (2048×), dropout in FC layers
  • SGD with momentum 0.9, batch size 128, L2 weight decay
  • 16.4% top-5 error on ImageNet

VGG (2015)

  • 19 layers, 144M parameters
  • Only 3×3 convolutions (stack to get larger receptive fields)
  • 6.8% top-5 error

YOLO (Object Detection)

  • Predicts bounding boxes and class probabilities on a 7×7 grid
  • Each grid cell predicts 2 boxes (x, y, h, w, confidence)
  • Single forward pass ⇒ real-time detection

U-Net (2016, Segmentation)

  • Encoder-decoder architecture with skip connections
  • Skip connections concatenate encoder features with decoder features at the same resolution
  • Widely used in medical imaging and scientific applications

Why CNNs Work Better Than FC Networks

Inductive Bias

CNNs have a better inductive bias (prior on the space of functions) for image-like data:

  • Weight sharing forces the network to process each location similarly
  • Local connectivity exploits spatial structure
  • Searches through a smaller family of functions, all of which are plausible for images
Result: far fewer parameters with better generalization. In MNIST-1D, a CNN with 2,050 parameters outperforms a FC network with 150,185 parameters.
