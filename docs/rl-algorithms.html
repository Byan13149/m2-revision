RL Algorithms

Lectures: L14–L15 • Prince: Chapter 19

Tabular RL

When $|S|$ and $|A|$ are small enough to store $v[s]$ or $q[s,a]$ in a table.

Dynamic Programming (Model Known)

Policy Iteration:

  1. Policy Evaluation: Compute $v[s]$ for current policy $\pi$ by sweeping through states and applying Bellman backups until convergence
  2. Policy Improvement: Greedily update $\pi(s) \leftarrow \arg\max_a q[s,a]$
  3. Repeat until policy stops changing

Value Iteration: Directly apply Bellman optimality:

$$v[s] \leftarrow \max_a \left( r[s,a] + \gamma \sum_{s'} \Pr(s'|s,a) \, v[s'] \right)$$

Simpler than policy iteration. The greedy policy at convergence is optimal.

Monte Carlo Methods (Model Unknown)

Estimate values by averaging complete returns from sampled episodes.

  • No bootstrapping (don't use value estimates in the update)
  • Pros: unbiased targets
  • Cons: high variance; must wait until episode ends

MC control loop:

  1. Generate episodes using current policy (with $\epsilon$-greedy for exploration)
  2. For each visited $(s,a)$, compute empirical return
  3. Update $q[s,a]$ as the average of observed returns
  4. Improve policy greedily

Temporal Difference (TD) Learning

Combines strengths of DP and MC: learns from experience (like MC) but bootstraps (like DP).

TD Error

$$\delta_t = r_{t+1} + \gamma \, v[s_{t+1}] - v[s_t]$$

Measures the discrepancy between the current estimate and the one-step-ahead estimate.

SARSA (On-Policy TD)

$$q[s_t, a_t] \leftarrow q[s_t, a_t] + \alpha \left( r_{t+1} + \gamma \, q[s_{t+1}, a_{t+1}] - q[s_t, a_t] \right)$$

Uses the actual next action $a_{t+1}$ sampled from $\pi$. Good when you care about safety during exploration.

Q-Learning (Off-Policy TD)

$$q[s_t, a_t] \leftarrow q[s_t, a_t] + \alpha \left( r_{t+1} + \gamma \max_{a'} q[s_{t+1}, a'] - q[s_t, a_t] \right)$$

Uses $\max_{a'}$ instead of the actual next action. The behavior policy can explore freely while learning the optimal $q$-function.

Deep RL: Function Approximation

When $|S|$ is too large for tables (e.g., images), approximate $q[s,a]$ with a neural network $q[s, a, \boldsymbol{\phi}]$.

Fitted Q-Learning

  1. Collect transitions: $\mathcal{D} = \{(s_t, a_t, r_{t+1}, s_{t+1})\}$
  2. Define target: $y_t = r_{t+1} + \gamma \max_{a'} q[s_{t+1}, a', \boldsymbol{\phi}]$
  3. Minimize: $\mathcal{L} = \sum_t (q[s_t, a_t, \boldsymbol{\phi}] - y_t)^2$
Instability: The Deadly Triad

Fitted Q-learning can be unstable due to three factors:

  1. Moving target: $y_t$ depends on $\boldsymbol{\phi}$, which keeps changing
  2. Correlated samples: consecutive transitions are similar
  3. Off-policy + function approximation + bootstrapping

Deep Q-Network (DQN)

Stabilizers that make deep Q-learning work:

  • Experience replay: store transitions in a buffer; sample random mini-batches (breaks correlation)
  • Target network: maintain frozen parameters $\boldsymbol{\phi}^-$ for computing targets; update periodically (stabilizes targets)
  • Frame stacking: stack recent frames to handle partial observability
  • $\epsilon$-greedy exploration

Architecture: a CNN maps stacked Atari frames to $q[s, a]$ for each discrete action.

Double DQN

Standard Q-learning overestimates values because $\max$ is biased. Fix: use one network to select the action, another to evaluate it:

$$y_t = r_{t+1} + \gamma \, q\!\left[s_{t+1}, \arg\max_{a'} q[s_{t+1}, a', \boldsymbol{\phi}], \boldsymbol{\phi}^-\right]$$

Policy Gradient Methods

Instead of learning $q$ and acting via argmax, directly optimize the policy $\pi[a \mid s, \boldsymbol{\theta}]$.

When to Use Policy Gradients

Value-based methods (Q-learning) require $\arg\max_a q(s,a)$, which is hard with continuous action spaces. Policy gradient methods work naturally with continuous actions.

REINFORCE

The simplest policy gradient algorithm. The gradient of the expected return is:

$$\nabla_\theta J(\theta) = \mathbb{E}_\pi \left[ \sum_t \nabla_\theta \log \pi[a_t \mid s_t, \theta] \cdot G_t \right]$$

Increase probability of actions that led to high returns; decrease for low returns.

Actor-Critic

Use a critic (value network) to reduce variance of the policy gradient:

  • Actor: policy network $\pi[a \mid s, \theta]$
  • Critic: value network $v[s, \mathbf{w}]$
  • Replace $G_t$ with the advantage: $A_t = G_t - v[s_t]$ (or use TD error)
