Neural Networks Review

Lectures: L01–L03 • Prince: Chapters 3–5

Notation (Prince Convention)

  • Input: $\mathbf{x} \in \mathbb{R}^{D_i}$, Output: $\mathbf{y} \in \mathbb{R}^{D_o}$
  • Parameters (weights + biases): $\boldsymbol{\phi}$
  • Model: $\mathbf{y} = f[\mathbf{x}, \boldsymbol{\phi}]$
  • Hidden units / pre-activations: values before the activation function
  • Activations: values after the activation function

Shallow Neural Networks (Ch 3)

A shallow network has one hidden layer:

$$f[\mathbf{x}, \boldsymbol{\phi}] = \boldsymbol{\beta}_0 + \boldsymbol{\Omega}_1 \cdot a\!\left(\boldsymbol{\theta}_0 + \boldsymbol{\Omega}_0 \mathbf{x}\right)$$

where $a(\cdot)$ is the activation function applied element-wise.

ReLU Activation

$$\text{ReLU}(z) = \max(0, z)$$

With ReLU, a shallow network computes a piecewise linear function. Each hidden unit contributes one "joint" (kink) to the output. With $D$ hidden units in 1D, you get at most $D$ joints.

Universal Approximation Theorem: A shallow network with enough hidden units can approximate any continuous function on a compact domain to arbitrary precision. However, "enough" may mean exponentially many units — depth helps.

TermMeaning
BiasesY-offsets (additive constants)

Deep Neural Networks (Ch 4)

Networks with more than one hidden layer. A deep network is a composition of simpler functions:

$$f[\mathbf{x}, \boldsymbol{\phi}] = f_K \circ f_{K-1} \circ \cdots \circ f_1(\mathbf{x})$$

where each $f_k$ is an affine transformation followed by an activation.

Why Depth Matters

Exponential Expressiveness

Deep networks with ReLU can represent exponentially more linear regions than shallow networks with the same number of parameters. A network with $K$ layers of width $D$ can produce $O(D^K)$ linear regions, whereas a shallow network with $KD$ units gives only $O(KD)$ regions.

Composing two shallow networks:

  • Network 1 produces a piecewise linear function with $D_1$ joints
  • Network 2 "folds" each linear piece of Network 1, multiplying the number of regions
  • Result: up to $D_1 \times D_2$ linear regions from just $D_1 + D_2$ hidden units

General Matrix Form

For a $K$-layer network:

$$\mathbf{h}_0 = \mathbf{x}$$ $$\mathbf{h}_k = a(\boldsymbol{\beta}_k + \boldsymbol{\Omega}_k \mathbf{h}_{k-1}), \quad k = 1, \ldots, K$$ $$\mathbf{y} = \boldsymbol{\beta}_{K+1} + \boldsymbol{\Omega}_{K+1} \mathbf{h}_K$$

Activation Functions

FunctionFormulaProperties
ReLU$\max(0, z)$Default choice. Sparse activations. Can have "dead neurons."
Leaky ReLU$\max(\alpha z, z)$Avoids dead neurons ($\alpha \approx 0.01$)
Sigmoid$\sigma(z) = 1/(1+e^{-z})$Output in $(0,1)$. Used for binary classification output.
Tanh$\tanh(z)$Output in $(-1,1)$. Zero-centered.
Softmax$e^{z_i} / \sum_j e^{z_j}$Output sums to 1. Used for multiclass classification output.
GELU$z \cdot \Phi(z)$Smooth approximation of ReLU. Used in transformers.

Forward and Backward Pass

Forward Pass

Compute activations layer by layer, from input to output. Store all intermediate values for the backward pass.

Backward Pass (Backpropagation)

Compute gradients of the loss $\mathcal{L}$ with respect to all parameters using the chain rule in reverse order.

$$\frac{\partial \mathcal{L}}{\partial \boldsymbol{\Omega}_k} = \frac{\partial \mathcal{L}}{\partial \mathbf{h}_k} \cdot \frac{\partial \mathbf{h}_k}{\partial \boldsymbol{\Omega}_k}$$
Backprop Complexity

One backward pass has roughly the same cost as one forward pass. Total gradient computation is $O(\text{number of parameters})$ per sample — this is what makes training neural networks tractable.

Automatic Differentiation

Backpropagation is a special case of reverse-mode automatic differentiation. PyTorch builds a computation graph during the forward pass, then traverses it in reverse to compute gradients exactly (not numerically).

Parameter Counting

For a fully-connected layer mapping $D_\text{in}$ inputs to $D_\text{out}$ outputs:

$$\text{Parameters} = D_\text{in} \times D_\text{out} + D_\text{out} = D_\text{out}(D_\text{in} + 1)$$

(weights + biases)

Common Exam Question

Given a network architecture, count the total number of trainable parameters. Remember to include biases and consider which layers have parameters (e.g., ReLU has none, BatchNorm has $2D$ learnable parameters).

Weight Initialization (Ch 7)

Proper initialization keeps activations and gradients in a well-scaled range throughout the network.

He (Kaiming) Initialization

For ReLU networks, initialize weights from:

$$w_{ij} \sim \mathcal{N}\!\left(0, \frac{2}{D_\text{in}}\right)$$

This compensates for the fact that ReLU kills half the activations (zeroes out negative values), so variance would otherwise shrink by half per layer.

Xavier (Glorot) Initialization

For tanh/sigmoid networks:

$$w_{ij} \sim \mathcal{N}\!\left(0, \frac{2}{D_\text{in} + D_\text{out}}\right)$$
