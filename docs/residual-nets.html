Residual Networks & Normalization

Lectures: L04–L05, L08 • Prince: Chapter 11

The Degradation Problem

Adding more layers to a plain (non-residual) network helps initially, but beyond a certain depth performance degrades — even on the training set. This is not overfitting; it's an optimization problem.

Shattered Gradients

In very deep networks, gradients at initialization change unpredictably across layers. With finite learning rates, these "shattered gradients" cause unstable training. The gradient signal becomes unreliable.

Residual (Skip) Connections

A residual block learns a residual function $g(\mathbf{h})$ and adds it to the input:

$$\mathbf{h}_{k+1} = \mathbf{h}_k + g[\mathbf{h}_k]$$

Each block only needs to predict the change from the previous layer, rather than the entire representation.

Why It Helps: Gradient Flow

  • Plain network gradient: $\frac{\partial \mathbf{h}_K}{\partial \mathbf{h}_k} = \prod_{l=k}^{K-1} \frac{\partial \mathbf{h}_{l+1}}{\partial \mathbf{h}_l}$ — products can vanish or explode
  • Residual network gradient: $\frac{\partial \mathbf{h}_K}{\partial \mathbf{h}_k} = \mathbf{I} + \ldots$ — always has an identity component, ensuring gradients can flow directly
Key insight: Residual connections roughly double the number of layers you can train before performance degrades. Combined with normalization, networks can go to hundreds of layers.

Batch Normalization

Normalizes activations within each mini-batch:

$$\hat{h}_j = \frac{h_j - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}$$ $$\tilde{h}_j = \gamma \hat{h}_j + \beta$$
  • $\mu_B, \sigma_B^2$: mean and variance computed over the mini-batch
  • $\gamma, \beta$: learnable scale and shift (initialized to 1 and 0)
  • In CNNs: statistics are computed over (batch) × (spatial dimensions) per channel
  • At test time: use running averages of $\mu$ and $\sigma^2$ collected during training

Effects: stabilizes training, smooths the loss surface, enables larger learning rates, acts as mild regularization.

Layer Normalization

Similar to BatchNorm, but normalizes over features/channels rather than over the batch:

  • Computes statistics per sample (not per batch)
  • Works with any batch size (including batch size 1)
  • Preferred in transformers and sequence models

ResNet Block Structure

A typical ResNet block:

x → Conv → BN → ReLU → Conv → BN → (+x) → ReLU

When dimensions change (e.g., downsampling), a 1×1 convolution is applied to the skip connection to match dimensions.

Effect on Loss Surfaces

Research shows that residual connections and batch normalization make loss surfaces dramatically smoother, with fewer sharp local minima. This enables stable training and better convergence.
