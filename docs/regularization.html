Regularization & Generalization

Lectures: L03 • Prince: Chapters 8–9

Bias-Variance Tradeoff (Ch 8)

Test error can be decomposed into three sources:

$$\mathbb{E}[\text{test error}] = \text{noise}^2 + \text{bias}^2 + \text{variance}$$
ComponentMeaningReduced by
NoiseInherent randomness in the data (irreducible)Better data collection
BiasSystematic error from model being too simpleMore capacity / flexibility
VarianceSensitivity to choice of training setMore data / regularization

Double Descent

Modern View: Double Descent

Classical theory predicts a U-shaped test error curve. But modern deep learning exhibits double descent: beyond the interpolation threshold (where the model can fit the training data perfectly), test error decreases again as model capacity increases further. This requires some form of regularization (even implicit) to work.

Choosing Hyperparameters

Use three data splits:

  • Training set: fit model parameters
  • Validation set: choose hyperparameters (capacity, learning rate, regularization strength)
  • Test set: evaluate final performance once

Explicit Regularization (Ch 9)

Add a penalty term to the loss:

$$\tilde{\mathcal{L}}[\boldsymbol{\phi}] = \mathcal{L}[\boldsymbol{\phi}] + \lambda \cdot \Omega[\boldsymbol{\phi}]$$

where $\lambda > 0$ controls regularization strength.

L2 Regularization (Weight Decay)

$$\Omega[\boldsymbol{\phi}] = \|\boldsymbol{\phi}\|_2^2 = \sum_j \phi_j^2$$
  • Encourages small weights ⇒ smoother functions
  • Also called Tikhonov regularization, ridge regression
  • In neural networks, usually applied to weights only (not biases) ⇒ "weight decay"

Probabilistic Interpretation

L2 regularization = Gaussian prior on weights. $$\Pr(\boldsymbol{\phi}) = \mathcal{N}(\mathbf{0}, \sigma_\phi^2 \mathbf{I})$$ MAP estimation with this prior gives the same objective as L2-regularized MLE, with $\lambda = 1/(2\sigma_\phi^2)$.

L1 Regularization

$$\Omega[\boldsymbol{\phi}] = \|\boldsymbol{\phi}\|_1 = \sum_j |\phi_j|$$

Encourages sparse weights (many exactly zero). Equivalent to a Laplace prior.

Implicit Regularization

SGD as Regularizer

  • Gradient descent with finite step size approximates a differential equation with an implicit regularization term
  • SGD noise disfavors regions where gradients are steep
  • SGD prefers solutions where all batches have similar gradients
  • Larger learning rates ⇒ stronger implicit regularization ⇒ often better generalization

Regularization Heuristics

Early Stopping

Stop training when validation loss starts increasing. Weights start small (near initialization) and don't have time to grow large and overfit. Reduces effective model complexity.

Ensembling

Train multiple models independently (different initializations or data subsets). Average their predictions. Reduces variance without increasing bias.

Dropout

During training, randomly zero out each hidden unit with probability $p$ (typically $p=0.5$). At test time, multiply weights by $(1-p)$.

  • Forces the network to not rely on any single hidden unit
  • Can be interpreted as training an implicit ensemble
  • Effectively reduces co-adaptation between neurons

Batch Normalization

Normalize activations within each mini-batch, then apply learnable scale and shift:

$$\hat{h}_j = \frac{h_j - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}, \quad \text{then} \quad \tilde{h}_j = \gamma \hat{h}_j + \beta$$

where $\mu_B, \sigma_B^2$ are batch statistics, and $\gamma, \beta$ are learned parameters. Typically initialized to $\gamma=1, \beta=0$.

Data Augmentation

Often the strongest regularizer for images. Apply label-preserving transformations:

  • Random flips, crops, rotations
  • Color jitter, scaling
  • For physics: physically-motivated transformations

Label Smoothing

Replace hard labels (0/1) with soft labels ($\epsilon / K$ for wrong classes, $1 - \epsilon + \epsilon/K$ for correct class). Prevents overconfident predictions.

Transfer Learning

Pre-train on a large dataset (e.g., ImageNet), then fine-tune on your smaller task-specific dataset. The pre-trained features act as a strong regularizer.
