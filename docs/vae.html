Variational Autoencoders

Lecture: L17 • Prince: Chapter 17

Latent Variable Models

The idea: model hidden causes $\mathbf{z}$ that explain the observed data $\mathbf{x}$.

  • Prior: $\Pr(\mathbf{z}) = \mathcal{N}(\mathbf{0}, \mathbf{I})$ (standard normal)
  • Decoder (generative model): $\Pr(\mathbf{x} \mid \mathbf{z}, \boldsymbol{\phi}) = \mathcal{N}(\mathbf{x}; f[\mathbf{z}, \boldsymbol{\phi}], \sigma^2\mathbf{I})$
  • Marginal likelihood: $\Pr(\mathbf{x} \mid \boldsymbol{\phi}) = \int \Pr(\mathbf{x} \mid \mathbf{z}, \boldsymbol{\phi}) \, \Pr(\mathbf{z}) \, d\mathbf{z}$
The Problem

The marginal likelihood integral is intractable for nonlinear decoders (neural networks). We cannot compute it exactly because the integral has no closed form.

The ELBO (Evidence Lower Bound)

Derivation Step 1: Introduce $q(\mathbf{z})$

Multiply and divide by an auxiliary distribution $q(\mathbf{z})$ inside the log-likelihood:

$$\log \Pr(\mathbf{x} \mid \boldsymbol{\phi}) = \log \int \frac{\Pr(\mathbf{x}, \mathbf{z} \mid \boldsymbol{\phi})}{q(\mathbf{z})} q(\mathbf{z}) \, d\mathbf{z}$$

Step 2: Apply Jensen's Inequality

Since $\log$ is concave, push it inside the expectation:

$$\log \Pr(\mathbf{x} \mid \boldsymbol{\phi}) \geq \mathbb{E}_{q(\mathbf{z})}\!\left[\log \frac{\Pr(\mathbf{x}, \mathbf{z} \mid \boldsymbol{\phi})}{q(\mathbf{z})}\right] = \text{ELBO}$$

The Gap: KL Divergence

$$\log \Pr(\mathbf{x}) = \text{ELBO} + D_\text{KL}(q(\mathbf{z}) \| \Pr(\mathbf{z} \mid \mathbf{x}))$$

Since KL $\geq 0$, ELBO $\leq \log \Pr(\mathbf{x})$. The bound is tight when $q(\mathbf{z}) = \Pr(\mathbf{z} \mid \mathbf{x})$ (the true posterior).

ELBO Decomposition

$$\text{ELBO} = \underbrace{\mathbb{E}_{q(\mathbf{z})}\left[\log \Pr(\mathbf{x} \mid \mathbf{z}, \boldsymbol{\phi})\right]}_{\text{Reconstruction term}} - \underbrace{D_\text{KL}(q(\mathbf{z}) \| \Pr(\mathbf{z}))}_{\text{KL to prior (regularization)}}$$
Two terms:
  1. Reconstruction: How well can we reconstruct $\mathbf{x}$ from samples of $\mathbf{z}$?
  2. KL regularization: How close is $q(\mathbf{z})$ to the prior $\mathcal{N}(\mathbf{0}, \mathbf{I})$?

VAE Architecture

Encoder (Inference Network)

$q(\mathbf{z} \mid \mathbf{x}, \boldsymbol{\theta}) = \mathcal{N}(\boldsymbol{\mu}_\theta(\mathbf{x}), \text{diag}(\boldsymbol{\sigma}_\theta^2(\mathbf{x})))$

A neural network that maps $\mathbf{x}$ to the parameters $\boldsymbol{\mu}$ and $\boldsymbol{\sigma}^2$ of the approximate posterior. This is amortized inference: one forward pass per sample.

Decoder (Generative Network)

$\Pr(\mathbf{x} \mid \mathbf{z}, \boldsymbol{\phi})$: a neural network that maps $\mathbf{z}$ to the parameters of $\Pr(\mathbf{x})$.

The Reparameterization Trick

Problem: The ELBO involves an expectation over $q(\mathbf{z} \mid \mathbf{x})$. We need gradients through the sampling operation, but sampling is not differentiable.

Solution: Separate the randomness from the deterministic computation:

$$\boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{I})$$ $$\mathbf{z} = \boldsymbol{\mu}_\theta(\mathbf{x}) + \boldsymbol{\sigma}_\theta(\mathbf{x}) \odot \boldsymbol{\epsilon}$$

Now $\mathbf{z}$ is a deterministic function of $\boldsymbol{\mu}, \boldsymbol{\sigma}, \boldsymbol{\epsilon}$, and we can backpropagate through $\boldsymbol{\mu}$ and $\boldsymbol{\sigma}$ to the encoder parameters.

Closed-Form KL Term

For two Gaussians (approximate posterior vs standard normal prior):

$$D_\text{KL}\!\left(\mathcal{N}(\boldsymbol{\mu}, \text{diag}(\boldsymbol{\sigma}^2)) \| \mathcal{N}(\mathbf{0}, \mathbf{I})\right) = \frac{1}{2}\sum_{j=1}^{d}\left(\mu_j^2 + \sigma_j^2 - \log \sigma_j^2 - 1\right)$$

VAE Training Algorithm

  1. Encode: $\boldsymbol{\mu}, \boldsymbol{\sigma}^2 = \text{Encoder}(\mathbf{x})$
  2. Sample: $\mathbf{z} = \boldsymbol{\mu} + \boldsymbol{\sigma} \odot \boldsymbol{\epsilon}$, where $\boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{I})$
  3. Decode: $\hat{\mathbf{x}} = \text{Decoder}(\mathbf{z})$
  4. Compute loss: $-\text{ELBO} = -\mathbb{E}[\log \Pr(\mathbf{x} \mid \mathbf{z})] + D_\text{KL}(q \| p)$
  5. Update: both encoder ($\boldsymbol{\theta}$) and decoder ($\boldsymbol{\phi}$) with Adam/SGD

In practice, the reconstruction term is estimated with a single sample of $\mathbf{z}$ per data point. The KL term is computed in closed form.

Optimization Perspective

  • Decoder updates improve model fit (better reconstruction)
  • Encoder updates tighten the bound (better approximation of true posterior)
  • Both are updated jointly in every mini-batch
